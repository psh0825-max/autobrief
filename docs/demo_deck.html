AutoBrief

The autonomous client-intake & proposal agent for a solo AI MVP studio.

Google ADK · Gemini on Vertex AI · MCP · Cloud Run

The problem

A one-person studio's revenue bottleneck isn't building — it's winning the work.

Every inbound inquiry = 3–5 hours of reading, researching, scoping, pricing, and proposal-writing before any contract. That caps the founder at ~3–4 serious inquiries a day.

AutoBrief, in one breath

A raw client email goes in. AutoBrief triages it, researches the client, prices it with a deterministic rubric, writes a proposal, and — only after a human approves — drafts the reply, calendar invite, Drive doc, and deck.

3–5 hours → ~1 minute of compute + a quick review.

Architecture

raw inquiry email │ RouterClassifier (flash, structured verdict) ├─ decline → DeclineAgent (polite reply) ├─ clarify → ClarifierAgent (3–5 questions) └─ proceed → AutoBriefPipeline (SequentialAgent) IntakeParser → Researcher[google_search] → ScoperEstimator → ProposalWriter → DeliveryAgent * (HITL-gated) * Drive · deck · calendar · Gmail draft — each needs human approval

State flows via output_key + instruction templating. Tiers: flash-lite · flash · pro.

The key idea: deterministic pricing

The LLM only classifies (archetype, feature add-ons, complexity, rush).

Every dollar and week is computed by a Python tool over rubric.yaml.

Quotes are reproducible and auditable — no hallucinated prices, ever.

Guardrails (verified)

🔒 No auto-send. Gmail only ever drafts; all 4 write tools each raise an adk_request_confirmation HITL gate. The read-only slot helper runs un-gated.
💲 Price-consistency. The proposal quotes the rubric's band verbatim.
🛡️ PII redaction before research · prompt-injection framing (email = untrusted data).
🔭 Observability. Cloud Run + Cloud Trace — every step, tool call, and HITL pause is traced.
